Let's discuss "var", "let" and "const"

In every programming language, there is a must common topic, variable. Variables are used to contain various types of values which can be modified or replaced with completely new values.

In Javascript, variables can be declared by three reserved keywords, such as "var", "let" and "const". Of them, "var" and "let" are of similar behavior, but the variables declared with "const" keyword has a different feature.

 "var" is a global scope variable declaration keyword. The variables declared with "var" can be accessed throughout the whole body of the document. Variables declared with "var" are hoisted, that means ES6 automatically moved up only the declaration part of the variable without the assigned value. So, if you want to access a variables value which is hoisted, before the declaration, you will get the value as undefined.

 "let" is a block scope variable declaration keyword. Variables declared with "let" keyword will only be available to accessed from the same block and inside blocks of it. Accessing let variables outside the scope will return an error.

 "const" is used to declare such a variable which value can not be completely changed. If the value of a const variable is of primitive data type like number, string, booleans etc., this will behave like a constant, not variable. But if the value of the variable is of non-primitive data types like arrays, objects, etc. then the value can be modified as these data types supports this features. "const" is also a block scope variable declaration keyword.

"let" and "const" variables are not hoisted, that means, when you declare variables using let and const keywords, ES6 never moves up the declaration part of the variable to the top of the document. Keep in mind that, only var variables are hoisted automatically.

Use cases of "null" and "undefined"

"null" and "undefined" are two different kind of data types in javascript. The variables containing the value "undefined" means that the variable has already been declared but no data was assigned to this variable yet. On the other hand, when we want to mean that a variable is declared and has a value assigned to it but the value is totally blank, we use "null" as the value of it.
